# planning/ - 任务管理区

> 本目录集中管理项目的任务规划、待办事项和变更记录。

## 目录结构

```
planning/
├── README.md          # 本文件（任务管理指南）
├── BACKLOG.md         # 待办事项池（未排期的任务）
├── SPRINT.md          # 当前迭代任务（正在进行的工作）
└── CHANGELOG.md       # 变更记录（已完成的变更）
```

## 何时读这里？

- **BACKLOG.md**：查看所有待办任务、添加新任务、评估优先级
- **SPRINT.md**：查看当前迭代目标、进行中的任务、阻塞项
- **CHANGELOG.md**：查看历史变更、追溯决策、了解演进

## 与 ROADMAP.md 的关系

| 文档 | 职责 | 粒度 |
|------|------|------|
| **ROADMAP.md** | 战略方向、里程碑、P0/P1/P2 优先级 | 粗粒度（周/月） |
| **BACKLOG.md** | 待办任务池、需求收集 | 细粒度（任务级） |
| **SPRINT.md** | 当前迭代、进行中任务 | 实时（日/周） |

## 任务生命周期

```
[新需求/Idea] → BACKLOG.md → [排期] → SPRINT.md → [完成] → CHANGELOG.md
                    ↑                                           │
                    └───────── [如需后续工作] ─────────────────┘
```
