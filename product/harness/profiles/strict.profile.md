# Strict Profile

This file extracts the stable staged execution profile from the legacy strict workflow prompt.

## Use this profile when

- the task has higher governance sensitivity or rework risk
- stage-by-stage evidence is required
- boundary expansion must be tightly controlled

## Stable profile rules

- freeze in-scope and out-of-scope before implementation
- produce auditable evidence at the end of every stage
- maintain explicit blocking-risk and rework-risk handling
- stop and escalate before expanding scope

## Minimum evidence set

- frozen execution contract
- per-stage completion and evidence summary
- gate result table
- residual risk and follow-up recommendations

## Escalation triggers

- an out-of-scope file becomes necessary
- a required gate cannot complete and no substitute evidence exists
- the task turns into a multi-task batching problem
