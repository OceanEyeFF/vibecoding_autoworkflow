# 测试证据技能

## 概览

本技能实现 `WorktrackScope.Verify` 算子的 `validation` lane（验证校验面），对应 Harness 控制回路中的**收集证据**阶段。它负责收集测试与验收维度的证据，为 `gate-skill` 的 validation-gate 提供输入。它与 `review-evidence-skill`（implementation lane）和 `rule-check-skill`（policy lane）共同构成 Verify 的三个正交校验面。

当 `Harness` 已经有一个活动中的 `工作追踪`，并需要一轮限定范围验证来为当前变更收集测试证据时，使用这个技能。

这个技能会为一轮 `通用高能力模型` `子代理` 打包最小验证上下文，把检查项映射到当前验收标准与验证要求，并返回一个供后续关卡判定使用的结构化验证维度封套。

## 何时使用

当当前问题不是"下一步应该构建什么"，而是"有哪些证据表明这项工作满足既定验收与验证门槛"时，使用这个技能：

- 当前工作项已经存在实现或内容变更
- `工作追踪约定` 定义了现在需要获得验证覆盖的验收标准
- 当前轮次包含显式的 `验证要求`
- 在进入关卡判定之前，`Harness` 需要新鲜或可复用的测试证据
- 下一步依赖于一份清晰说明：哪些内容已被验证、哪些失败、哪些仍未覆盖

## 工作流

1. 载入当前验证轮所需的最小 `工作追踪范围` 产物。
2. 为限定范围的 `通用高能力模型` `子代理` 构建一份 `测试证据任务简报` 和一份 `测试证据信息包`。
3. 为本轮建立验证映射：
   - 哪些验收标准在范围内
   - 哪些验证要求适用
   - 哪些检查、命令或现有结果可以满足它们
4. 在复用前，先把现有验证证据分类为 `新鲜`、`复用但新鲜`、`混合`、`过期` 或 `未知`。
5. 只针对本轮收集或刷新限定范围验证证据：
   - 当已有结果已经满足要求时，复用新鲜且可信的结果
   - 当要求仍缺乏证据时，运行恰好缺失的检查
   - 当所需证据无法安全产出时，将要求标记为被阻塞或未覆盖
6. 应用验证分诊，让失败、阻塞或未覆盖的要求始终保持显式；只有狭窄的低严重度残留才允许折叠进 `残留风险`。
7. 产出一份固定格式的 `验证证据报告`。
8. 在进入审查综合、关卡判定、恢复规划或收尾之前停止。

## 硬约束

- 本技能是控制回路的证据收集层（validation lane）；负责收集测试、验收条件与运行结果的验证证据；验证证据的输出只能是测试结果与覆盖状态，最终裁决与状态更新的行为禁止出现在验证证据的输出中。
- 唯一合法行为是引用当前 `工作追踪约定` 中已定义的验收标准、排除目标与验证要求；重定义这些标准的行为必须返回 blocked。
- 充分证据必须来自可复现的自动化检查、明确关联验收标准的测试结果或可验证的命令输出；自称完成的手动检查、一个绿色差异、或模糊的信心表述本身不能作为充分证据出现在输出中。
- 唯一合法行为是对缺失证据返回显式的 `未覆盖` 或 `被阻塞` 状态；把缺失证据转换成隐式通过的行为必须返回 blocked。
- 本技能的输出只能是测试维度的验证证据；代码审查综合、策略检查或最终关卡判定的行为禁止出现在本技能的输出中。
- 当限定范围验证包已经足够时，唯一合法行为是只传递最小验证包；传入过宽的代码仓库上下文的行为必须显式暴露并说明超出范围的必要性。
- 唯一合法行为是收集并返回验证证据；变更 `Harness 控制状态`、关闭工作追踪，或直接建议合并的行为必须返回 blocked 并路由到对应控制算子。
- 唯一合法行为是将失败或不稳定的检查显式暴露在返回的证据表面中；隐藏失败或不稳定检查的行为必须返回 blocked。
- 唯一合法行为是复用新鲜度依据明确标记且未过期的验证证据；新鲜度依据不明确的验证证据必须被标记为 `未知` 并禁止复用。
- `残留风险` 的输出只能容纳低严重度容忍项；失败检查、被阻塞要求或未覆盖要求必须显式出现在 `未覆盖或被阻塞项` 中，其行为禁止被吸收到 `残留风险` 中。
- 当结果主要由不稳定或受环境约束的证据构成时，唯一合法行为是报告 `低置信度` 并显式说明不稳定性来源；报告高置信度的行为必须返回 blocked。

## 预期输出

使用这个技能时，产出一份至少包含以下章节的 `验证证据报告`：

- `验证目标`
- `验收覆盖`
- `验证执行`
- `时效性与复用依据`
- `已收集证据`
- `未覆盖或被阻塞项`
- `置信度与残留风险`
- `返回 Harness`

- `子代理模型`
- `维度编号`
- `证据轮次`
- `验证目标`
- `当前工作项`

- `维度判定结果`
- `已检查验收标准`

- `覆盖摘要`
- `已运行检查`
- `已复用检查`
- `分检查结果`
- `证据路径或命令`
- `置信度`
- `置信度理由`
- `残留风险`
- `已应用低严重度吸收`
- `低严重度吸收理由`
