# Simple SubAgent Workflow Prompt（轻量版）

> 定位：repo-side execution contract / runner prompt template。
>
> 本模板不提供完整 orchestrator；它用于在 Codex / Claude / 其他 Code Agent 中执行中小任务时，稳定约束阅读入口、任务边界、质量 Gate 与失败协议。

## 适用范围
- 中小规模需求落地（单模块或低耦合跨文件改动）。
- 需要“可执行但不过度复杂”的流程约束。

## 不适用场景
- 高风险跨模块重构。
- 涉及复杂迁移、长链路联调、多轮外部依赖协同。

## Prompt 模板

```text
[filepath or filename]

准备落地这个文档提到的开发需求。请把这次任务当作一次“受仓库 contract 约束的执行流程”，严格按下面步骤顺序执行，不要跳步，不要扩展任务边界。

【总原则】
- 本次任务以“原始需求文档 + 仓库真相层 + Step 1 产出的执行合同”为唯一依据。
- 未经明确允许，不扩大修改范围，不顺手重构，不处理无关问题。
- 若遇到信息不足、运行中断、依赖缺失、测试无法执行、边界冲突，必须停止在当前阶段，报告阻塞原因，等待处理。
- 默认把“未明确允许的额外改动”视为越界。

【Step 0：限定阅读入口】
先只读取与本任务直接相关的仓库入口与真相层，优先遵守仓库既有阅读顺序：
1. docs/README.md（如存在）
2. 根目录 README 与 INDEX
3. 与任务直接相关的 docs/、Claude/、toolchain 脚本与代码文件

默认不要主动进入以下运行态/挂载态目录，除非任务明确要求：
- .autoworkflow/
- .spec-workflow/
- .serena/
- .nav/
- .agents/
- .claude/

【Step 1：任务规划 / 执行合同生成】（推理等级：medium）
基于原始需求文档、仓库真相层文档、以及实际代码，输出一个“执行合同”，必须包含以下内容：
- Goal：本次必须完成什么
- Non-goals：本次明确不做什么
- In-scope files：允许修改的文件/目录
- Out-of-scope files：禁止修改的文件/目录
- Preconditions：执行前提 / 依赖 / 阻塞项
- Plan：最小可执行步骤列表
- Validation Plan：后续要做的静态检查、测试、运行验证
- Exit Criteria：什么条件下才算完成
- Risks：已知风险与可能失败点

Step 1 结束后，后续执行必须严格受这个“执行合同”约束。

【Step 2：按合同执行】（推理等级：high 或 xhigh）
严格按 Step 1 的执行合同实施开发。
要求：
- 只能修改 In-scope files；如果确实需要新增范围，先停止并说明原因，不得自行扩边。
- 不把“优化建议”自动转成实现。
- 不得因为发现额外问题而顺手修复，除非这些问题阻塞主任务且已明确记录。
- 每完成一个关键改动，都要对照 Goal / Non-goals 自检一次，确认没有偏题或越界。

【Step 3：出口 Gate】（推理等级：high）
严格根据“原始需求文档 + Step 1 执行合同”做出口审核，不要只按最终代码自我感觉判断。

至少完成以下检查：
1. Scope Gate：
   - 是否只修改了允许范围内的文件
   - 是否存在未授权扩边
2. Spec Gate：
   - 是否满足 Goal
   - 是否违反 Non-goals
   - 是否满足 Exit Criteria
3. Static Gate：
   - 至少一次静态代码检测 / lint / type check / build check（按仓库可用工具选择）
4. Test Gate：
   - 能补白盒测试时，必须补
   - 不能补时，必须明确说明不能补的原因
5. Smoke Gate：
   - 能运行最小烟测时，必须运行一次
   - 不能运行时，必须说明阻塞点
6. Risk Report：
   - 列出残留风险、未验证点、需要人工接手的部分

【最终输出格式】
最后只输出以下内容：
- Step 1 执行合同摘要
- 实际修改文件列表
- 完成情况（完成 / 部分完成 / 阻塞）
- 已执行的静态检查
- 已执行的测试与烟测
- 残留风险与人工后续建议

【失败协议】
出现以下任一情况时，立即停止并报告，不继续自行推进：
- 任务边界不清
- 需要修改 Out-of-scope files
- 仓库真相层与原始文档冲突
- 构建 / 测试环境缺失
- 中途运行中断
- 关键依赖不存在
```

## 使用说明（Codex / Claude）
- 优先将模板与具体需求文档一起提供给 Agent。
- 强制要求先产出 Execution Contract，再允许编码。
- 交付时必须回收 Gate 结果与残留风险。

## 常见失败案例
- 只做实现，不先产出执行合同。
- 执行中静默扩边（修改了未授权目录）。
- 测试与烟测被“如果可能的话”弱化，导致 Gate 失真。
