# docs/knowledge/ - 知识库文档

> 背景知识、分析洞察、参考资料的沉淀区。

## 何时读我？

- ✅ 需要理解项目演进历史
- ✅ 需要查阅最佳实践和参考资料
- ✅ 需要复盘和问题分析
- ❌ 不在执行具体任务时阅读

## 子目录

| 目录 | 职责 | 何时读 |
|------|------|--------|
| [analysis/](analysis/) | 分析洞察、失败模式复盘 | 需要改进抓手时 |
| [guides/](guides/) | 用户指南、最佳实践 | 需要操作指导时 |
| [reference/](reference/) | 参考资料、外部知识 | 需要背景知识时 |

## 与其他层的关系

本目录是**横向支撑**，不属于 overview → modules → interfaces 的纵向细化链路。

```
overview/  modules/  interfaces/
    ↑          ↑          ↑
    └──────────┴──────────┘
              ↓
         knowledge/ (本目录)
```
