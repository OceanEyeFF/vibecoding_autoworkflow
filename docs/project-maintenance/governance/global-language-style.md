# 全局语言风格

> 目的：固定跨任务的人读输出默认口径，优先服务判断、执行和收口，而不是背景铺垫。

## 一、默认原则

- 先给结论，再给解释。
- 前 20% 内容必须明确两件事：
  - 最终判断是什么
  - 当前是否可直接执行
- 推理、补充说明和背景放在最后；没有额外决策价值时可以省略。
- 前半部分不写背景复述、横向方案对比或长列表分析。

## 二、默认结构

默认按下面结构组织：

```text
【TL;DR】
一句话结论 + 是否可执行

【Key Points】
- 最多 3 条关键点

【Action】
- 最多 3 条可执行动作

【Optional Reasoning】
- 可选；不影响理解与执行
```

## 三、使用约束

- `TL;DR` 必须让读者不看后文也能做判断。
- `Key Points` 只保留会影响决策的最小信息，不展开背景。
- `Action` 只写当前应做的动作；没有动作时明确写不可执行或待补条件。
- `Optional Reasoning` 只承接推理、取舍和补充证据，不反向改写前面的结论。

## 四、判断标准

如果下面几句话成立，说明输出风格合格：

- 读者在开头就知道结论和可执行性
- 不需要先读完背景，才能知道该不该做
- 推理被后置后，主结论仍然完整可用
