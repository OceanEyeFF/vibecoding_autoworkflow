## 版本目标

验证 AutoDev 核心工作流是否可行，建立用户信心。

## 功能范围

### ✅ 包含功能

| 功能模块 | 详细说明 |
|---------|---------|
| **Phase 1: 需求精炼** | AskUserQuestion 澄清需求 + G1 门禁检查 |
| **Phase 2: 任务规划** | 项目结构分析 + 任务拆分（3-7个）+ TodoWrite 任务列表 + G2 门禁 |
| **Phase 3: 迭代开发** | 逐任务执行 + Level 0 即时修复（≤3次）+ G3 门禁 |
| **Phase 4: 验收提交** | 完整测试 + 交付报告 + G4 门禁 + Git 提交（用户确认） |
| **Level 0 回路** | 测试失败时即时修复，最多 3 次尝试 |

### ❌ 不包含功能

- Git 检查点回退（v0.2）
- Level 1 任务重构回路（v0.2）
- Level 2 整体重规划回路（v0.3）
- 状态持久化 .autodev/state.json（v0.2）
- 指令模块化 phases/gates/loops/（v0.3）

## 状态管理

v0.1 仅使用 **TodoWrite（内存）** 管理任务状态：

```javascript
TodoWrite([
  {
    content: "任务1: 实现登录UI",
    status: "completed",
    activeForm: "实现登录UI"
  },
  {
    content: "任务2: 实现登录API调用",
    status: "in_progress",
    activeForm: "实现登录API调用"
  },
  {
    content: "任务3: 实现登录状态管理",
    status: "pending",
    activeForm: "实现登录状态管理"
  }
])
```

**局限性**：
- ❌ 无法记录 retry 次数
- ❌ 无法记录 Git commit 信息
- ⚠️ 中断恢复能力有限（只能靠 TodoWrite 状态推断）

## 门禁检查清单

### G1: 需求完整性

- [ ] 验收标准已明确（≥3 条，可测试）
- [ ] 功能范围已界定（有"不做什么"）
- [ ] 核心价值已清晰（能一句话解释）
- [ ] 用户已确认需求理解

### G2: 任务可执行性

- [ ] 任务数量在 3-7 个之间
- [ ] 每个任务有完成标准
- [ ] 验证命令已确定（如 `npm test`）
- [ ] 用户已确认任务规划

### G3: 测试通过

- [ ] 单个任务的测试全部通过
- 如失败：
  - retry ≤ 3 → Level 0 即时修复
  - retry > 3 → 人工介入

### G4: 交付完整性

- [ ] 所有验收标准已满足
- [ ] 完整测试通过（`npm test`）
- [ ] 交付报告已生成
- [ ] 用户确认可以提交

## Level 0 回路详解

**触发条件**：G3 门禁失败（测试不通过）

**执行流程**：

```
retry = 0
   ↓
┌──────────────────────────────┐
│ 1. 分析错误信息              │
│    - 读取完整错误堆栈        │
│    - 定位失败的具体测试      │
│                              │
│ 2. 定位问题原因              │
│    - Read 相关代码文件       │
│    - 理解预期 vs 实际行为    │
│                              │
│ 3. 修复代码                  │
│    - 使用 Edit 工具修改      │
│    - 保持小步修改（≤10行）   │
│                              │
│ 4. 重新测试                  │
│    - Bash("npm test")        │
│    - retry++                 │
│                              │
│ 5. 判断结果                  │
│    ├─ ✅ 通过 → 任务完成     │
│    └─ ❌ 失败 → retry ≤ 3?   │
│              ├─ Yes → 回步骤1│
│              └─ No  → 人工介入│
└──────────────────────────────┘
```

**人工介入输出格式**：

```markdown
## 🚨 调试受阻（Level 0 失败）

**任务**：任务2 - 实现登录API调用

**问题**：axios.post 返回 404 错误

**已尝试**：
1. 检查 API endpoint 路径 - 结果：路径正确
2. 检查请求参数格式 - 结果：格式正确
3. 查看后端日志 - 结果：后端未收到请求

**需要主人帮助**：
🔸 提供更多上下文（后端是否启动？）
🔸 调整任务目标（先实现 mock API？）
🔸 跳过此任务，先处理其他任务
```

## 验收标准（v0.1 本身）

根据迭代计划，v0.1 需要满足：

- [x] 能完整走通 Phase 1 → 2 → 3 → 4
- [x] G1/G2/G3/G4 门禁能正确检查
- [x] Level 0 循环能工作
- [x] 失败 3 次后能请求人工介入
- [x] TodoWrite 状态管理可用
- [x] 代码修改工具使用正确（Read/Write/Edit/Bash）

## 已知限制

1. **状态管理简陋**：仅用 TodoWrite，无法持久化
2. **无 Git 回退**：Level 0 失败后只能人工介入，无法自动回退
3. **无任务重构**：无法自动调整任务粒度
4. **中断恢复有限**：只能靠 TodoWrite 状态推断，不可靠
5. **retry 计数不持久**：无法在对话中断后恢复 retry 次数

**这些限制将在 v0.2、v0.3、v1.0 中逐步解决。**
