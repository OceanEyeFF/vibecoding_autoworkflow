# AI高效使用指南

## 1. 如何更高效地使用AI

### 1.1 Step 1 生成 SystemPrompt「个人指令包」（10 min）

#### 1.1.1 简单写一份你给AI的名片

你的AI名片应包含以下内容：
1. 我的技术栈，AI要掌握的技术栈
2. 我需要 AI 做什么，不要做什么
3. 必须遵守的约束
4. 输出格式（如果需要）

#### 1.1.2 【AI 反向 Prompt】你现在是“Prompt 优化师”

将你的AI名片交给AI（把你写好的整块 Prompt 直接丢给任意大模型），对AI的结果按你的需要实时反馈迭代出“你的专属 AI 名片”。

#### 1.1.3 每一个专家都需要专属的SystemPrompt「个人指令包」

如果你要处理不同方向的事务，那么每一个方向（每一个专家）都需要专属的SystemPrompt，而不是用一个通用提示试图覆盖所有领域。

不同领域AI的差异：
- 知识聚焦不同
- 同一术语差异化
- 行为风格差异化
- 约束条件不同

#### 1.1.4 示例：资深安卓开发工程师的SystemPrompt

```
1. 你是一名资深安卓开发工程师，使用 Kotlin 编写所有代码；
2. 使用 Kotlin 语言实现所有功能；
3. 不使用 XML 或 Jetpack Compose，所有 UI 布局都需使用 Kotlin 代码构建；
4. 如需异步操作或网络请求，优先使用 Kotlin 协程 (Coroutines)；
5. 可以在一段逻辑上添加注释备注功能用途，以及代码逻辑，但不需要每几行就添加注释；
6. 如果有多个实现方案，可以列举出来并对比优缺点，以及推荐最优方案；
```

### 1.2 Step 2 绑定编译器，植入“全局 System Prompt”，实现“一键插入”（5 min）

#### ChatGPT 官方客户端
- Settings → Custom Instructions → 粘贴 my_sys.txt 到“System”栏
- 在底部自带的CodeX中绑定你的编译器，允许对你当前打开的工程的修改

#### Cursor
- .cursorrules 文件里直接写 system: <你的AI名片>

#### VS Code
- 下载 VS Code Continue 插件
- 在 config.json 里添加：
  ```json
  "systemMessage": "<你的AI名片>"
  ```

#### Android Studio
- 下载CodeX插件并绑定你的Token

### 1.3 Step 3 提问（1~3 min）

#### 基本提问（1min）
此时，你的AI小助理几乎已经可以做到随意提问。可以达到90%以上在第一轮问答中解决你的问题了，3轮问答之内满足你的所有想象。

#### 高级提问（3min）
知道如何向AI提问可以解决你剩余9.9999999999999999999999999999%的问题

向AI提问是一门艺术！具体技巧可以通过向任意AI询问 "我是XXXX(XX开发工程师)，我该如何向AI提问" 来学习，并将对AI提问的模板套路熟练起来，相信你一定都会成为了不起的AI驾驭者。
